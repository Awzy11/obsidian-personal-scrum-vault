# Contributing

Thanks for wanting to improve this vault. Contributions are welcome — this is a living system and there's real value in how different people adapt it.

## What's worth contributing

- **Template improvements** — better structure, clearer prompts, missing sections in `90 - Templates/`
- **Workflow documentation** — clearer explanations in `80 - Reference/`, especially for edge cases
- **Claude Code / AI setup** — corrections or additions to `80 - Reference/Claude Code Setup.md`, especially for Windows (WSL) or Mac-specific steps
- **ADHD adaptations** — additional strategies that work well with the system
- **Bug fixes** — broken links, incorrect instructions, formatting issues

## What's out of scope

- Personal notes, daily notes, or any actual vault content (only the template structure lives here)
- Plugins — this vault intentionally requires zero community plugins to function
- Overhauls to the core workflow — open an issue to discuss before a large structural change

## How to contribute

1. Fork the repo
2. Make your changes on a branch
3. Open a pull request with a short description of what you changed and why
4. Keep PRs focused — one change per PR is easier to review than a batch

## Opening issues

If something is unclear, broken, or missing — open an issue. You don't need a fix ready. A clear problem statement is enough to start.
